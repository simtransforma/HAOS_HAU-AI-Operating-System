# PRE_EXEC_CHECKLIST.md

Use este checklist antes de iniciar qualquer tarefa operacional.

- [ ] Pedido classificado como tarefa operacional
- [ ] Fluxo definido como HAOS-first
- [ ] Conselho acionado
- [ ] Mega-Brain acionado (síntese executiva)
- [ ] Diretor de Projeto acionado
- [ ] Estrategista acionado
- [ ] Squad paralela definida
- [ ] Delegação aplicada (main como orquestrador)
- [ ] Skill aplicável selecionada
- [ ] Plano de validação final definido
- [ ] Plano de reporte por etapa definido (quem reporta, quando, formato)
- [ ] Formato de reporte validado: [etapa][agente][ação][evidência][status/bloqueio]
- [ ] Regra anti-silêncio ativa (update a cada <= 3 min)
- [ ] Plano de contingência para agente sem resposta definido (timeout + re-disparo + escalonamento)
- [ ] Escopo confirmado (IN/OUT, sem desvio)
- [ ] Responsável por etapa definido

## Regra de bloqueio

Se qualquer item ficar pendente, **não executar**.
Responder ao usuário com bloqueio + motivo + próximo passo.
