# OpenClaw Xtreme2 — Documentação Completa do Sistema

## 1) Visão geral
`HAOS_HAU-AI-Operating-System` é o repositório operacional com governança HAOS, automação de fluxo, agentes especialistas e evidência obrigatória.

## 2) Modelo operacional oficial
**Diretriz oficial:** CORE HAOS executa. Mega-brain é referência e não substitui o rito.

## 3) Fluxo HAOS completo (obrigatório)
**ABERTURA → CONSELHO-Fase1 → REPORT-SOLICITANTE → CONSELHO-Fase2 → MEGA_BRAIN → DIRETOR → ESTRATEGISTA → EXECUCAO → VALIDACAO → (se reprovado: CONSELHO_SE_REPROVADO [máx 3] → MEGA_BRAIN) → CONSELHO_Final_Aprovado → ENTREGA → REGISTRO**

### Definições curtas
- **ABERTURA:** entrada formal da tarefa com escopo e objetivo.
- **CONSELHO-Fase1:** decisão de direção, prioridade e risco.
- **CONSELHO-Fase2:** decisão de viabilidade e abordagem técnica.
- **REPORT-SOLICITANTE:** plano aprovado com tarefas, donos e sequência.
- **EXECUÇÃO:** implementação do plano por squad responsável.
- **VALIDAÇÃO:** conferência de qualidade e aceite.
- **REGISTRO:** fechamento com evidências e conclusão oficial.

## 4) Formato obrigatório de reporte
`[etapa][agente][ação][evidência][status/bloqueio]`

## 5) Papéis
- **Main/Orquestrador:** coordena e delega; sem execução solo.
- **CONSELHO-Fase1:** delibera decisões de negócio.
- **CONSELHO-Fase2:** delibera decisões técnicas.
- **Squad especialista:** executa e evidencia.
- **QA/Review:** valida antes do registro.
